# Contributing to Swarm Commander

Thank you for your interest in contributing to Swarm Commander! This project aims to make parallel AI coordination simple and accessible to everyone.

## Ways to Contribute

### 1. Share Your Experience

- **Use case templates:** Created a template for a specific domain? Share it!
- **Success stories:** Document how you used Swarm Commander
- **Workflow improvements:** Discovered a better way to coordinate?

### 2. Improve Documentation

- Fix typos and unclear explanations
- Add examples and use cases
- Translate to other languages
- Improve the Quick Start guide

### 3. Enhance Templates

- Simplify existing templates
- Add domain-specific templates
- Create starter kits for common project types

### 4. Report Issues

- Unclear documentation
- Template problems
- Missing features
- Edge cases not covered

## How to Contribute

### For Documentation & Templates

1. Fork the repository
2. Create a branch: `git checkout -b improve-docs`
3. Make your changes
4. Submit a Pull Request

### For Use Case Examples

1. Create a new folder in `examples/`
2. Include:
   - `README.md` explaining the use case
   - Sample order files
   - Sample Command Center
3. Submit a Pull Request

### For Issues

1. Check existing issues first
2. Use the issue template if provided
3. Include:
   - What you expected
   - What happened
   - Steps to reproduce

## Guidelines

### Template Changes

When modifying templates:
- Keep them simple - complexity defeats the purpose
- Maintain backwards compatibility when possible
- Document breaking changes clearly
- Include examples of usage

### Documentation

- Write for someone new to the concept
- Use concrete examples
- Keep language simple and direct
- Include both "what" and "why"

### Code of Conduct

- Be respectful and constructive
- Welcome newcomers
- Focus on the work, not the person
- Assume good intentions

## Pull Request Process

1. Update relevant documentation
2. Add your changes to the CHANGELOG if significant
3. Ensure your contribution follows existing style
4. Request review from maintainers

## Questions?

- Open a GitHub Discussion for general questions
- Open an Issue for bugs or feature requests
- Tag maintainers if urgent

## Recognition

Contributors will be recognized in:
- The README acknowledgments section
- Release notes for significant contributions
- The contributors graph

---

Thank you for helping make AI coordination better for everyone!
